写博客

语法

参数

layout代表的所建文章的位置，默认post

布局 路径
post source/_posts
page source
draft source/_drafts
自定义路径
参数 描述
-p, --path 自定义新文章的路径
-r, --replace 如果存在同名文章，将其替换
-s, --slug 文章的 Slug，作为新文章的文件名和发布后的 URL

博文创建方法

  1. 直接创建：
    • hexo new 测试一 / hexo new “测试一”
    • 在博客目录 下的source文件夹下的__post文件夹下创建名测试一.md文件；
    • 创建一个 source/_post/测试一.md 文件，同时 Front Matter 中的 title 为 "测试一me";
  2. 在_post下创建文件夹路径
    • hexo new page -p 测试三/me 或者 hexo new page -P 测试三/Me “title”
    • Hexo 会在source的_posts文件夹下新创建测试三文件夹并在此下创建me.md;
    • source/_posts/测试三/me.md，同时 me.md 的 Front Matter 中的 title 为 "page";
  3. 创建页面
    • hexo new page 测试二 “测试二”
    • 新建页面的方法;
    • 在博客目录 下的source文件夹下创建的测试二文件夹并在其下创建名index.md文件;
    • 创建一个 source/测试二/index.md 文件，同时 Front Matter 中的 title 为 "测试二me"；

Butterfly部分

安装以及应用

  1. 在Hexo根目录下

  2. 升级方法

  3. 应用

    修改Hexo根目录下的__config.yml,把主体改为butterfly

  4. 安装插件

  5. 注意个人建议

    在 hexo 的根目录创建一个文件 _config.butterfly.yml，并把主题目录的 _config.yml 内容复制到 _config.butterfly.yml 去。( 注意: 复制的是主题的 _config.yml ,而不是 hexo 的 _config.yml)

    注意：

    • 不要把主题目录的 _config.yml 删掉
    • 以后只需要在 _config.butterfly.yml进行配置就行。
      如果使用了 _config.butterfly.yml， 配置主题的 _config.yml 将不会有效果。
    • Hexo会自动合并主题中的_config.yml和 _config.butterfly.yml里的配置，如果存在同名配置，会使用_config.butterfly.yml的配置，其优先度较高。

博客工具

图床工具

图床工具:自动把本地图片转换成链接的一款工具，PicGo 就是一款比较优秀的图床工具。

  1. 方便自己在写博客的时候图片的快速引用以及上传
  2. 可以家快网页加载速度(因为项目过大会导致加载速度慢)
  3. 配置图床工具Hexo+Typora+Picgo+Gitee(码云),请到
  4. 如果不显示图片请把图片压缩到1M以下

Github Action

Github Action就是持续集成（Continuous Integration）」、「持续交付（Continuous Delivery）」、「持续部署（Continuous Deployment）」，简单来说就是Github出的自动部署的工具；

  1. 可以回退到某个版本,算是一个版本工具
  2. 自动部署工具

Gulp 压缩博客静态资源

  1. 前端构建工具。
  2. 项目开发完成，部署到线上，为了加快网站访问速度，会将如javascript/html/image/css进行合并压缩。
  3. 现在将这些操作编写成一个一个的任务。想做某个操作时，在命令行中操作，执行一个命令。任务就自动执行。如，敲一个命令，css代码自动压缩。

以上工具都应该在设置butterfly主题后设置
